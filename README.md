# Betting Flow E2E

Python QA automation framework for the Bet Placement flow. The project combines a focused Selenium/Pytest E2E test, a Requests-based API validation test, lightweight API contract checks, structured test data, and browser failure artifacts.

## Quick Start

```bash
make install
make test
```

## Stack

- Python 3.12+
- Pytest
- Selenium WebDriver with desktop Chrome
- Requests for API checks
- JSON Schema checks
- Ruff for formatting and lint checks
- Pytest HTML and JUnit reporting
- Allure for graphical reports

## Configuration

The assignment base URL defaults to the submitted test application. The user ID can be provided through environment variables or Pytest CLI options.

Use `.env.example` if local settings need to be overridden:

```bash
cp .env.example .env
set -a
source .env
set +a
```

Install Allure to view reports:
```bash
brew install allure
```

Common environment variables:

```bash
export USER_ID="<your-assignment-user-id>"
export BASE_URL="https://qae-assignment-tau.vercel.app"
```


## Formatting

Quality checks validate formatting, linting, and syntax compilation:

```bash
make check
```

## Run Tests

Run the full automated suite:

```bash
make test
```

Run only API tests:

```bash
make api
```

Run only UI tests:

```bash
make ui
```

## Reports and Artifacts

There are two report paths:

- HTML/JUnit: simple files, no external viewer required, generated by `make test`
  under `reports/report.html` and `reports/junit.xml`.
- Allure: richer step-by-step report with labels, parameters, and attachments,
  generated by `make allure` under `reports/allure-results/`.

**Allure**

Create report:
```bash
make allure
```

View the Allure report locally. This requires the Allure CLI; install by running `brew install allure`.
```bash
make allure-open
```

## CI

GitHub Actions is configured in `.github/workflows/tests.yml`.

- Push and pull request trigger API and UI tests.
- CI uploads HTML/JUnit/Allure reports and browser artifacts.

## Documentation

Supporting QA deliverables are in `docs/`:

- `test_plan.md`: prioritized manual scenarios and expected results.
- `execution_results.md`: executed checks with pass/fail status and linked defects.
- `bug_reports.md`: product bugs with concise reproduction steps and evidence.
- `strategy_recommendations.md`: focused quality strategy and release recommendations.

## Structure

```text
pyproject.toml           # Python target, dependencies, Ruff, pytest config
requirements.txt         # Editable install entry point for the project and dev tools
Makefile                 # Optional repeatable quality and test commands
conftest.py              # Pytest fixtures and custom CLI options
.github/workflows/       # CI for quality checks, API test, and UI smoke

src/betting_flow_e2e/
  api/
    client.py            # Thin requests-based API wrapper
    assertions.py        # Reusable API assertions
    contracts/
      assertions.py      # JSON parsing and schema validation helpers
      openapi.json       # Focused OpenAPI response-schema slice
  config.py              # Environment + pytest CLI driven settings
  data/
    test_data.py         # Successful UI journey data
  domain/
    betting.py           # Bet selection and payout helpers
    matches.py           # Small shared match helpers
  reporting/artifacts.py # Browser failure evidence capture
  ui/
    driver_factory.py    # Local/remote WebDriver creation
    base_page.py         # Shared Selenium waits/actions
    selectors.py         # Data-testid-first selector targets
    assertions.py        # Grouped receipt assertions
    pages/               # Full page objects
    components/          # Page component objects
    flows/               # High-level business actions
  utils/money.py         # EUR parsing/formatting helpers

tests/
  api/                   # API business-rule tests
  ui/                    # E2E UI browser tests

docs/
  test_plan.md                   # Manual/risk-based test plan
  execution_results.md           # Executed checks and pass/fail results
  bug_reports.md                 # Product defects with evidence
  strategy_recommendations.md    # Focused quality strategy and recommendations
```
